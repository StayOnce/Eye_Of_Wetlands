## 📖 项目简介

**“湿地之眼”** 是一套基于多源大数据的湖泊生态质量与公众感知协同分析系统。针对传统湖泊评价体系重客观指标、轻公众感知的痛点，本研究融合**遥感大数据（GEE）** 与**社交媒体文本（UGC）**，构建了生态质量指数（EI）与公众感知指数（PI），并引入耦合协调度模型（CCD）揭示武汉市10个典型湖泊在2021–2025年间的“自然—社会”协同演化规律。

系统提供**全局概览**、**单湖监测**、**耦合协调分析**、**滞后预警**及**大模型智能诊断**五大功能模块，将复杂的多维数据转化为可视化决策支持，推动湖泊治理从“单一生态管控”向“生态—感知协同提升”转型。

## ✨ 核心特色

- **双指数耦合评价体系**：综合遥感指标（MNDWI/NDVI/NDTI）与BERT/LDA情感主题模型，构建EI与PI指数，弥补主客观评价割裂的缺陷。
- **四类湖泊协同模式**：基于耦合协调度D值将湖泊划分为协同均衡型、感知领先型、双重滞后型、生态领先型，支持分类施策。
- **时空演化全景可视**：时序折线图、箱线图、EI-PI散点图、空间分布地图联动，展示湖泊生态与感知的梯度格局（“近郊优，城区弱”）。
- **智能诊断与预警**：集成滞后预警列表（基于EI/PI变化率），并接入**腾讯云DeepSeek-R1大模型**，支持自然语言提问（如“墨水湖生态现状如何？”），自动结合数据生成归因分析与修复建议。
- **交互式可视化大屏**：基于Leaflet+ECharts实现地图标记、图表联动、下钻分析，手机端自适应。

# 🚀 快速开始

## 1. 克隆仓库

```bash
git clone https://github.com/your-username/wetland-eye.git
cd wetland-eye
```

## 2. 创建虚拟环境（推荐）

```bash
python -m venv venv
source venv/bin/activate      # Linux/Mac
venv\Scripts\activate         # Windows
```

## 3. 安装依赖

```bash
pip install -r requirements.txt
```

注意：`requirements.txt` 已包含 Flask、pandas、numpy 及 tencentcloud-sdk-python。如需单独安装腾讯云SDK：`pip install tencentcloud-sdk-python`。

## 4. 配置大模型密钥（可选，若不配置聊天功能会返回错误）

系统通过腾讯云DeepSeek-R1模型提供智能诊断。你需要准备：

- 腾讯云账号 (开通大模型服务)
- 获取 SecretId 和 SecretKey

**方式一：环境变量 (推荐)**

```bash
# Linux / Mac
export TENCENT_SECRET_ID="你的SecretId"
export TENCENT_SECRET_KEY="你的SecretKey"

# Windows (cmd)
set TENCENT_SECRET_ID=你的SecretId
set TENCENT_SECRET_KEY=你的SecretKey
```

**方式二：临时测试**  
若无密钥，系统将返回“未配置密钥”错误，但其余所有可视化功能（地图、图表、预警）均可正常使用。

## 5. 运行应用

```bash
python app.py
```

应用默认运行在 `http://0.0.0.0:5000`。打开浏览器访问即可。

# 📊 数据说明

`data/lakes_data.json` 包含以下核心内容：

- **quarters**：20个季度（2021Q1 ~ 2025Q4）
- **lakes**：10个湖泊的元数据
  - `name`：湖泊名
  - `type`：耦合模式 (I区高生态-高感知 / II区低生态-高感知 / III区低生态-低感知 / IV区高生态-低感知)
  - `ei` / `pi` / `d`：最新季度的生态质量指数、公众感知指数、耦合协调度
  - `coord`：[经度, 纬度] (用于地图标记)
  - `ei_series` / `pi_series`：每个湖泊20个季度的EI/PI时序数据

🔬 **数据来源**：EI基于GEE提取MNDWI/NDVI/NDTI并经熵权法计算；PI基于多平台社交媒体（小红书/微博/抖音/知乎/携程）评论文本，经BERT情感分析和LDA主题聚类得到。

# 🗺 功能模块与接口

## 前端交互模块

| 模块 | 功能描述 |
|------|----------|
| 全局概览 | 展示所有湖泊EI/PI/D均值，EI箱线图（季度分布），EI/PI时序折线图（多湖对比） |
| 空间分布地图 | 湖泊标记大小映射D值、颜色映射EI值；点击标记切换至单湖模式 |
| 单湖监测 | 高亮单个湖泊的EI/PI时序曲线，更新KPI卡片为单湖数值，底部场景栏显示趋势评估 |
| 耦合协调分析 | EI-PI散点图，点大小表示D值，颜色区分四类耦合模式 |
| 滞后预警列表 | 基于最近两个季度变化率，自动生成“生态退化预警/协调度偏低/感知异常上升”等信息 |
| 智能诊断助手 | 用户输入自然语言问题，系统自动匹配上下文（如“东湖近两年生态趋势”），调用大模型生成回答 |

## 后端API接口

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/lakes` | GET | 返回所有湖泊当前EI/PI/D及坐标 |
| `/api/ei_timeseries` | GET | 返回所有湖泊EI时序数据 |
| `/api/pi_timeseries` | GET | 返回所有湖泊PI时序数据 |
| `/api/boxplot` | GET | 返回EI箱线图数据（按季度聚合） |
| `/api/ccd_scatter` | GET | 返回当前EI-PI散点数据 |
| `/api/warnings` | GET | 返回滞后预警列表 |
| `/api/lake_detail/<lake_name>` | GET | 返回单个湖泊详细时序趋势 |
| `/api/chat` | POST | 大模型对话 (需配置密钥) |

# 🎯 使用示例

- **查看全局生态格局**：页面加载即展示10个湖泊的EI/PI均值、地图色块、箱线图。
- **分析单个湖泊**：点击地图上的标记（如“东湖”），右侧图表自动高亮其曲线，KPI切换为该湖数据，底部显示趋势判断。
- **识别滞后风险**：打开“智能诊断”面板，查看预警列表（例如“墨水湖 · 生态退化预警”）。
- **自然语言提问**：在诊断面板输入“汤逊湖的公众感知为什么那么高？”，系统会结合湖泊类型、PI时序、空间服务补偿等因素给出分析。

# 🔬 研究支撑简述

本系统严格对应竞赛论文《“湿地之眼”：基于多源大数据的武汉湖泊生态质量—公众感知耦合与协同分析》的核心结论：

- **生态质量呈阶梯化分层**：优质稳定型（梁子湖/涨渡湖/汤逊湖）、中位敏感型（东湖/后官湖等）、低质脆弱型（墨水湖/沉湖）。
- **公众感知呈四个维度**：生态健康、休闲体验、景观审美、自然保护；满意度呈圈层递减（远郊高、城区低）。
- **耦合协调驱动机制**：随机森林表明“自然基底定上限 (NDVI/NDTI贡献超45%)，社会感知调幅度”。
- **四类耦合模式**：协同均衡型、感知领先型、双重滞后型、生态领先型。

系统可视化全面复现了论文中的图3~图18，并增加了大模型交互创新。
